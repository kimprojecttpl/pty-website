## เกี่ยวกับบริษัท (Company)

หจก. พัฒนายนตร์ (Patanayont) คือผู้จัดจำหน่ายอะไหล่รถยนต์แท้สำหรับรถญี่ปุ่นชั้นนำ ก่อตั้งขึ้นเมื่อปี **พ.ศ. 2507** ดำเนินธุรกิจมากว่า **60 ปี** เริ่มต้นจากการนำเข้าอะไหล่รถยนต์จากประเทศญี่ปุ่น และได้รับความไว้วางใจจากร้านค้าอะไหล่รถยนต์ทั้งค้าส่งและค้าปลีกทั่วประเทศ ด้วยปรัชญาธุรกิจที่ **ซื่อสัตย์ต่อลูกค้า** มาโดยตลอด

บริษัทมุ่งเน้นจัดจำหน่ายเฉพาะอะไหล่รถยนต์แท้ (Genuine Parts) และอะไหล่คุณภาพ OES (Original Equipment Service) วางตำแหน่งเป็น **ศูนย์อะไหล่รถยนต์แบบ ONE-STOP** ที่พร้อมให้บริการครบวงจร ผ่านบุคลากรมืออาชีพ คลังสินค้าและศูนย์กระจายสินค้า นวัตกรรมเทคโนโลยีที่ทันสมัย และระบบมาตรฐาน ISO

ดำเนินงานภายใต้กลุ่มบริษัทพัฒนายนตร์ที่มี **3 สำนักงาน** ได้แก่ สำนักงานสวนมะลิ สำนักงานสุขุมวิท109 และสำนักงานกลการ

### ประวัติ (History)

หจก. พัฒนายนตร์ ก่อตั้งขึ้นเมื่อปี **พ.ศ. 2507 (ค.ศ. 1964)** โดยเริ่มกิจการจากการนำเข้าอะไหล่รถยนต์จากประเทศญี่ปุ่น ต่อมาได้รับความไว้วางใจจากร้านค้าอะไหล่รถยนต์ทั้งค้าส่งและปลีก ในการกระจายอะไหล่รถยนต์คุณภาพสูง และให้บริการแก่ผู้ใช้รถยนต์ทั่วประเทศ มาเป็นเวลากว่า 60 ปี จนกลายเป็นบริษัทที่มีชื่อเสียงยาวนานในวงการอะไหล่รถยนต์ของไทย

### Dark Header Gradient (สัญลักษณ์ของแบรนด์)

นี่คือ dark chrome เบื้องหลัง top header, main nav และ map_header — ไล่จากเทาบนลงถ่านล่าง กรอบด้วยเส้นส้มพัฒนายนตร์ บน-ล่าง

```css
/* Canonical Patanayont header gradient — gray top → charcoal bottom */
background: linear-gradient(
  to bottom,
  rgba(120,119,119,1) 0%,   /* #787878 — header gray  */
  rgba(64,64,64,1)   51%,   /* #404040 — charcoal mid */
  rgba(65,65,65,1)  100%    /* #414141 — charcoal end */
);
border-top: 1px solid #f39200;     /* Patanayont Orange */
border-bottom: 1px solid #f39200;  /* map_header ใช้ 2px #f6921e แทน */
```

> ลำดับสต็อป **เทาบน → ถ่านล่าง** คือทิศทาง gradient มาตรฐานของแบรนด์ (ยืนยันซ้ำจาก IE fallback `startColorstr='#787777' → endColorstr='#414141'`)

> [!IMPORTANT]
> **เว็บไซต์จริงใส่ฟิลเตอร์ขาวดำทับอยู่ — สีจึงดู "หม่น" ไม่ใช่สีจริงของแบรนด์**
> ใน `style.css` มี `html { filter: grayscale(85%); -webkit-filter: grayscale(85%); }` (บรรทัด 8–11; ตัว `filter` อยู่บรรทัด 9–10) ซึ่งลดความอิ่มสีของทั้งหน้าจอลง 85% **ที่ตอน render เท่านั้น** เกือบแน่นอนว่าเป็น gesture **ไว้อาลัย (mourning)** (มี popup `princesspassedaway.jpg` ประกอบ) ค่าสี hex/rgb ที่เก็บใน CSS คือ **สีแบรนด์จริงที่อิ่มเต็ม** ไม่ใช่สีเทา
> **วิธีคืนสีจริง:** ลบหรือคอมเมนต์บรรทัด `filter: grayscale(...)` ออก (`responsive.css` มี `filter:none` คืนสีในบาง breakpoint อยู่แล้ว ยืนยันว่าเป็น overlay ที่ปิดเปิดได้ ไม่ใช่เจตนาดีไซน์)

> **Normalization notes สำหรับ redesign:**
> - **Orange** มี 4 เฉดเกือบเหมือน — ใช้ `#f39200` เป็น **THE brand orange** และ `#f6921e` เป็น variant อ่อนที่ยอมรับได้ ส่วน `#f69320` / `rgb(246,146,31)` คือ drift จากการ encode ให้ยุบรวมเหลือ 2 เฉด
> - **Charcoal** `#404040` กับ `#414141` ต่างกัน 1 หน่วย RGB แยกแทบไม่ออก ให้ยุบเป็นโทเคนเดียว `#404040`
> - **Accent discipline:** เขียว `#7eda7e` กับเหลือง `#ffcc00` ใช้อย่างละ 1 ครั้งโดยตั้งใจ ห้ามเลื่อนขึ้นเป็นสีหลัก — เก็บไว้เป็น highlight เฉพาะจุด

### Accessibility — สิ่งที่ต้องระวัง

- **ส้มบนขาวไม่ผ่าน WCAG:** `#f39200` บน `#ffffff` มี contrast เพียง **2.35:1** (ไม่ผ่าน AA สำหรับ body text ที่ต้องการ 4.5:1 และไม่ผ่าน 3:1 สำหรับ large text) → **อย่า** ใช้ส้มเป็นตัวอักษรเนื้อหา/ตัวเล็กบนพื้นขาว ใช้กับหัวข้อใหญ่ (24px+ bold), เส้นขอบ, ไอคอน, พื้นปุ่มเท่านั้น
- **ส้มที่ผ่านมาตรฐาน:** `#f39200` บน `#404040`/ดำ = **8.92:1** (ผ่าน AAA) — ถ้าต้องการ "ตัวอักษรสีส้ม" ให้วางบน dark chrome
- **ปุ่มส้ม:** ขาวบนส้ม `#f39200` ก็ได้แค่ 2.35:1 → ใช้ตัวอักษร **ดำ/ถ่าน** บนปุ่มส้ม หรือสงวนขาว-บน-ส้มไว้เฉพาะ label ใหญ่ตัวหนา
- **เทา:** ขาวบน header `#404040` = **10.37:1** (AAA, ดีเยี่ยม) · `#787878` บนขาว = **4.42:1** (เฉียดไม่ผ่าน AA body, ใช้กับ large text/border/UI รองได้) → body copy ให้ใช้ `#5f5f5f` หรือเข้มกว่า; `#333` บนขาว ≈ **12.6:1** (AAA)

**Kanit** เป็นฟอนต์หลักของทุก UI ทั้งไทยและละติน ตั้งบน body เป็น `font-family: 'Kanit', sans-serif` ใช้กับหัวข้อ (h4 section titles), เนื้อหา (render น้ำหนักบาง 100/300 ให้หน้า about/product ดูโปร่ง), nav links, สโลแกน ONE-STOP และปุ่มปิด popup ตัวอักษร geometric โค้งมนเล็กน้อยของ Kanit คือฐานที่ palette ทั้งหมดถูกออกแบบให้วางอยู่ใต้

**Source Sans Pro** เป็นฟอนต์ละตินสำรอง โหลดคู่กับ Kanit ไว้ backstop การ render ตัวอักษรละติน/ฟอร์ม (ไม่มี `font-family` กำหนดตรงๆ ใน `style.css`)

### Grayscale-filter fix (คืนสีจริงของแบรนด์)

เว็บไซต์ปัจจุบันใส่ overlay ไว้อาลัยทับทั้งหน้า ทำให้สีดูหม่น — **คอมเมนต์หรือลบบรรทัดนี้ออก** เพื่อคืนสีแบรนด์เต็มความอิ่ม

```css
/* style.css บรรทัด 8–11 — REMOVE / COMMENT OUT to restore true brand colors
html {
  filter: grayscale(85%);
  -webkit-filter: grayscale(85%);
}
*/

/* หรือบังคับปิด overlay ทั้งระบบ */
html {
  filter: none !important;
  -webkit-filter: none !important;
}
```

> หมายเหตุ: เก็บ snippet `grayscale(85%)` ไว้ใช้ซ้ำได้ในอนาคต (เช่นช่วงไว้อาลัยระดับชาติ) โดยทำเป็น toggle class แทนการ hard-code บน `html` — `responsive.css` มีตัวอย่าง `filter:none` คืนสีบาง breakpoint อยู่แล้ว ยืนยันว่ามันเป็น overlay ที่เปิด-ปิดได้ ไม่ใช่เจตนาดีไซน์ของแบรนด์
